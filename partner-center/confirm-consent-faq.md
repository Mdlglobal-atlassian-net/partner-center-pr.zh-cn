## <a href="" id="processfaq"></a>处理

**确认客户接受 Microsoft 云协议的新过程是什么？**

Microsoft 致力于帮助我们的客户和合作伙伴保持兼容。 作为 Microsoft 的云解决方案提供商 (CSP) 计划中的合作伙伴，你负责查看 Microsoft 的条款和条件，包括 Microsoft 产品和服务与客户的可接受用法。 更好地帮助合作伙伴满足合规性要求并确保透明度，我们会启用一个新的过程，需要以确认其客户已接受 Microsoft 云协议条款的云解决方案提供商合作伙伴。 从 2018 年 8 月 7 日，我们引入了新字段在合作伙伴中心仪表板用户界面中也适用于的合作伙伴中心 API 合作伙伴提供此信息。 最初它将在可选提供此确认，但它需要将从 2018 年 11 月 7 日开始。 

**此要求时将生效？**

合作伙伴可能会开始确认接受，只要在 2018 年 8 月 7 日功能才可用。 从 2018 年 11 月 7 日，合作伙伴中心仪表板用户需要确认客户接受 Microsoft 云协议上所有新的交易。 2019 年 3 月 22 日合作伙伴中心 API 的用户将需要执行相同操作。 API 用户的截止时间已扩展，以允许它们在其他时间的技术实现。

**新 |Microsoft 是否验证所提供的合作伙伴的客户信息的准确性？**

Microsoft 验证提供的客户电子邮件地址属于正确的格式。 它是合作伙伴的责任提供准确的信息并更正任何发现不准确的信息。 

**新 |如何可以合作伙伴证明客户接受 Microsoft 云协议，无论它签名，记录，数字化，或在纸张上？**

合作伙伴负责确保客户已接受 Microsoft 云协议的条款，用于管理其自己的进程和记录。 通过使用合作伙伴中心仪表板或合作伙伴中心 API 过程，合作伙伴确认客户已接受 Microsoft 云协议的条款。

**新 |Microsoft 如何检查确认过程已经完成了客户？**

从开始 7，2018 年 11 月，当云解决方案提供商合作伙伴启动事务为新的或现有客户使用合作伙伴中心仪表板，Microsoft 的系统将自动检查客户确认信息是否存在于合作伙伴中心仪表板。 如果不存在，交易不能才能完成。 

从开始 3 月 22，2019，当云解决方案提供商合作伙伴启动新的或现有客户使用合作伙伴中心 API 的事务，Microsoft 的系统将自动检查客户确认信息是否存在。 如果不存在，交易不能才能完成。 

**新 |是向客户发送通知的确认过程中提供其电子邮件地址？**

没有通知的进程。

## <a href="" id="policyfaq"></a>策略

**Microsoft 要求合作伙伴以确认客户已接受 Microsoft 云协议的为什么？** 

Microsoft 云协议合作伙伴、 客户和 Microsoft 通过提供保护客户的使用权限、 条款和条件授予 Microsoft 云产品和服务订单。 

Microsoft 云协议的正确执行保护我们的合作伙伴，客户和 Microsoft 通过确保针对多个重要的主题包括相互对齐，但不是限于安全、 隐私和数据保护。 确认过程可确保透明度和对齐方式，并允许 Microsoft 以更好地帮助客户需要立即响应法规的查询。

**哪些客户信息完成所需的确认过程？**
有关接受此协议的人员是必需的以下信息：
-   协议接受的日期
-   第一个和最后一个名称
-   电子邮件地址
-   电话号码 （可选） 

**新 |如果我们不能满足到期日期，会发生什么情况？**

合作伙伴可能会开始确认接受，只要在 2018 年 8 月 7 日功能才可用。 从 2018 年 11 月 7 日，合作伙伴中心仪表板用户需要确认客户接受 Microsoft 云协议上所有新的交易。 2019 年 3 月 22 日合作伙伴中心 API 的用户将需要执行相同操作。

在强制执行时，为现有客户或更改席位计数现有订阅的新购买不允许将确认客户接受之前。 确认接受之前，不会为新客户，允许交易记录。 此要求不会影响自动续订。

**此要求适用于在哪个区域？**

需要 Microsoft 公共云下的所有地区的 Microsoft 云协议客户接受确认。 （包括用于美国政府和 Microsoft 云德国 Microsoft 云） 的国家云是不受支持的客户接受确认。

**哪些合作伙伴负责确认到 Microsoft 云协议客户接受？**

2018 年 11 月 7 日，云服务提供商 (CSP) 的间接提供商、 云解决方案提供商直接计费的合作伙伴，这两者均负责确认客户接受和可以使用合作伙伴中心 API 或合作伙伴中心仪表板来执行此操作生效。 间接提供商可能需要先从其间接经销商收集所需的信息，然后将其提供给 Microsoft 通过合作伙伴中心 API 或合作伙伴中心仪表板。 通过合约，它是间接提供商具有协定与 Microsoft 以确保其云解决方案提供商经销商的最终客户已接受到 Microsoft 云协议的绑定。 

**哪些客户方案需要确认客户接受 Microsoft 云协议？**

云解决方案提供商合作伙伴希望以下交易记录，则必须确认接受 Microsoft 云协议：
1.  新客户完成新购买
2.  现有客户完成新购买
3.  现有客户更新到现有订阅的席位数

**可以确认为独立于其新购买的现有客户接受或席位新增内容？**

是。 客户接受可以确认为 Microsoft 云协议涵盖的所有客户。

**对于具有多个云解决方案提供商合作伙伴的客户要求是什么？**

确认接受的合作伙伴确认。 如果客户而且交易与多个云解决方案提供商合作伙伴，每个合作伙伴将需要确认接受。

**我云解决方案提供商间接经销商与云解决方案提供商间接提供商合作。 我的责任是什么？**

通过合约，它是间接提供商具有协定与 Microsoft 以确保最终客户已接受 Microsoft 云协议的绑定。 但是，间接提供商可能有与间接经销商的单独排列收集客户接受。 在此情况下，间接提供商将需要首先从其间接经销商收集所需的客户接受信息，然后将其提供给 Microsoft 通过合作伙伴中心 API 或合作伙伴中心仪表板。 你可以通过立即向你根据其请求的间接提供商提供所需的信息来支持此过程。

**我云解决方案提供商间接提供商。 是否需要与我的云解决方案提供商间接经销商要准备进行协调？**

需要从其间接经销商收集所需的客户接受信息的间接提供商建议通信到其间接经销商此要求立即以协调传递和收到此信息。

**我如何可以解释为我的客户，它们应确认其接受 Microsoft 云协议？**

Microsoft 云协议的正确执行保护 Microsoft、 合作伙伴和客户通过确保针对多个重要的主题包括相互对齐，但不是限于安全、 隐私和数据保护。 确认过程可确保透明度和对齐方式，并允许 Microsoft 以更好地帮助客户需要立即响应法规的查询。

**新 |如果客户适用于单个的云解决方案提供商间接提供商，但事务完成与多个云解决方案提供商间接经销商，是否每个云解决方案提供商间接经销商需要云解决方案提供商间接提供商提供的确认信息？**

云解决方案提供商间接提供商负责向 Microsoft 提供的 Microsoft 云协议客户接受确认。 对于每个客户帐户，可以一次在合作伙伴中心仪表板中记录只有一个确认。 

云解决方案提供商间接提供商负责确认负责同意 Microsoft 云协议客户联系人已完成操作，而不考虑云解决方案提供商间接经销商联系他们与之交易的数量。

**新 |谁应该是以确认接受 Microsoft 云协议客户组织内的授权的人员？**

它是从客户组织内人员获取客户接受 Microsoft 云协议合作伙伴的责任。 它是确定谁有权提供验收客户的责任。 

**新 |哪个版本的 Microsoft 云协议是合作伙伴负责确认客户接受？ 合作伙伴需要指示哪个版本的 Microsoft 云协议客户已接受吗？**

合作伙伴负责进行云解决方案提供商交易记录与新客户时使用最新的 Microsoft 云协议。 合作伙伴必须确认接受的最新的 Microsoft 云协议时续订、 更改或完成与现有客户的新交易。 

在之前续订，将更新为 Microsoft 云协议的情况下，合作伙伴不需要重新立即获取客户接受 Microsoft 云协议客户，但必须重新确认同意之前的 Microsoft 云协议为续订。 但是，你可能想要让客户接受新条款之前续订利用 Microsoft 云协议中的新增和改进的条款。

**新 |我的云解决方案提供商间接经销商。 如果我在我的公司的记录中记录客户接受 Microsoft 云协议，是否需要还提供此信息通过合作伙伴中心仪表板或合作伙伴中心 API？**

通过合约，它是间接提供商具有协定与 Microsoft 以确保最终客户已接受 Microsoft 云协议的绑定。 但是，间接提供商可能有与间接经销商的单独排列收集客户接受。 在此情况下，间接提供商将需要首先从其间接经销商收集所需的客户接受信息，然后将其提供给 Microsoft 通过合作伙伴中心 API 或合作伙伴中心仪表板。

**新 |如果我的客户不一致的 Microsoft 云协议条款？**

作为 Microsoft 的云解决方案提供商计划中的合作伙伴，你负责为客户接受 Microsoft 云协议之前订单。 这不是新的要求。 出于任何原因，如果客户不会再同意的 Microsoft 云协议，然后合作伙伴必须取消与此客户的订阅。

**新 |经销商可以提供代表客户同意？**

不需要。

**一旦确定客户提供了验收，是否需要重新确认客户将来？**

一次确认，客户已接受 Microsoft 云协议在合作伙伴中心中的客户帐户部分将保留此信息，不需要重新确认的持续时间涵盖客户接受协议的条款。 仅必须重新确认验收之前放置新订单。

在之前续订，将更新为 Microsoft 云协议的情况下，合作伙伴不需要重新立即获取客户接受 Microsoft 云协议客户，但必须重新确认同意之前的 Microsoft 云协议为续订。 但是，你可能想要让客户接受新条款之前续订利用 Microsoft 云协议中的新增和改进的条款。

**新 |我不舒适向 Microsoft 提供我最终客户的直接联系人信息。 Microsoft 如何使用此信息？**

合作伙伴可以创建客户租户，时，需要为客户提供主要联系人信息。 每在线服务条款，"Microsoft 将处理个人数据仅在从客户的记录说明上。"

## <a href="" id="technicalfaq"></a>技术

**如何合作伙伴确认客户接受并向 Microsoft 提供？ 可用于支持此过程的工具？**

从 2018 年 8 月 7 日开始提供的两种方法来帮助合作伙伴： 
1.  合作伙伴可以使用合作伙伴中心 API 以编程方式确认客户接受。 
2.  合作伙伴可以使用合作伙伴中心仪表板确认客户接受。

**新 |我使用合作伙伴中心 API 的云解决方案提供商合作伙伴。 哪些 API 更改是否需要考虑？** 

在此情况下，需要考虑 API 更改是：

-   [https://docs.microsoft.com/en-us/partner-center/develop/get-confirmation-of-customer-consent](https://docs.microsoft.com/en-us/partner-center/develop/get-confirmation-of-customer-consent) 

-   [https://docs.microsoft.com/en-us/partner-center/develop/get-agreement-metadata](https://docs.microsoft.com/en-us/partner-center/develop/get-agreement-metadata)

**如果有困难实现合作伙伴中心 API？**

可以通过在合作伙伴中心仪表板手动确认接受。

**新 |作为使用合作伙伴中心仪表板或合作伙伴中心 API 的替代项，可以我批量上传文件与客户确认信息改为？**

此时，在合作伙伴中心仪表板和合作伙伴中心 API 是提供的 Microsoft 云协议客户接受确认的唯一接受的方法。

**新 |我使用第三方平台或"中间"件执行功能将我的公司的系统连接的合作伙伴中心的合作伙伴。 我需要执行哪些操作？**

合作伙伴使用平台集成合作伙伴中心 API 的建议直接咨询有关其计划和时间线的提供程序的集成自动化客户确认过程的合作伙伴中心 API 功能。

**新 |你可以为导入客户完整名称和电子邮件地址，以及 Microsoft 云协议接受日期提供 API 流程示例？**

合作伙伴建议查看详细信息和示例的合作伙伴中心 API 文档。

**新 |如何获取最新的 Microsoft 云协议，以向客户显示，并在确认记录用作我 API 过程的一部分？**

当前合作伙伴中心 API 非常简单，并且不提供合作伙伴可以以编程方式检索一份 Microsoft 云协议，也不会捕获版本的 Microsoft 云协议客户接受确认的一部分。 

**新 |对于此过程使用合作伙伴中心 API 需要用户 Id 参数。 当通过不与 Azure AD 集成的第三方平台订单，如何管理此要求？**

用户 Id 在本例中标识可提供的客户同意确认合作伙伴租户下的员工。 如果员工没有相应的 Azure AD 用户帐户下的合作伙伴租户，则建议您为该目的指定特定的用户帐户并创建一个过程来单独捕获的员工提供标识确认。 

**新 |Microsoft 要求合作伙伴确认而不是直接从客户的 Microsoft 云协议客户同意的为什么？**

客户从云解决方案提供商合作伙伴，未直接从 Microsoft 购买。 使用服务，Microsoft 提供的 Microsoft 提供的条款，会记录在 Microsoft 云协议。 Microsoft 云经销商协议需要能够查看并接受 Microsoft 云协议之前订单的客户的云解决方案提供商合作伙伴。 

**新 |我使用 API 和合作伙伴中心仪表板 web 用户界面来确认 Microsoft 云协议客户接受。 后 2018 年 11 月 7 日，什么是以确认客户接受我选项？**

这两个选项是可用和可接受的确认 Microsoft 云协议客户接受。 使用合作伙伴中心仪表板的交易记录将需要确认要存在从 2018 年 11 月 7 日。 使用合作伙伴中心 API 的交易记录将需要确认要存在开始 2019 年 3 月 22 日。
