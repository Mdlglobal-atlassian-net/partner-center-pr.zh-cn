## <a href="" id="processfaq"></a>正在

**确认客户接受 Microsoft 云协议的新过程是什么？**

Microsoft 致力于帮助我们的客户和合作伙伴保持合规。 作为 Microsoft 云解决方案提供商 (CSP) 计划中的合作伙伴, 你负责审查 Microsoft 的条款和条件, 包括可接受的 Microsoft 产品和服务对客户的使用。 为了更好地帮助合作伙伴满足合规性要求并确保透明性, 我们正在启用一个新过程, 要求 CSP 合作伙伴确认其客户已接受 Microsoft 云协议条款。 截至2018年8月7日, 我们在合作伙伴中心仪表板用户界面中引入了一个新的字段, 合作伙伴中心 API 还提供了此信息。 最初, 提供此确认将是可选的, 但从2018年11月7日开始需要此确认。 

**此要求何时生效？**

此功能将于 2018 年 8 月 7 日推出，合作伙伴随即可以开始进行接受确认。 从2018年11月7日开始, 合作伙伴中心仪表板用户需要确认客户接受所有新事务的 Microsoft 云协议。 2019年3月22日, 将要求合作伙伴中心 API 用户执行相同操作。 API 用户的截止时间已延长, 使技术实现更有时间。

**新建 |Microsoft 是否验证合作伙伴提供的客户信息的准确性？**

Microsoft 验证提供的客户电子邮件地址的格式是否正确。 合作伙伴负责提供准确的信息并纠正发现的任何不准确的信息。 

**新建 |合作伙伴如何证实客户接受 Microsoft 云协议, 无论是签署、记录还是数字化, 还是在纸张上？**

合作伙伴负责确保客户已接受 Microsoft 云协议的条款以及管理自己的流程和记录。 通过使用合作伙伴中心仪表板或合作伙伴中心 API 流程, 合作伙伴确认客户已接受 Microsoft 云协议的条款。

**新建 |Microsoft 将如何检查客户的确认过程是否已完成？**

从2018年11月7日开始, 当 CSP 合作伙伴使用合作伙伴中心仪表板为新客户或现有客户启动事务时, Microsoft 系统将自动检查合作伙伴中心是否存在客户确认信息。板. 如果不存在，该交易将无法完成。 

从2019年3月22日开始, 当 CSP 合作伙伴使用合作伙伴中心 API 为新客户或现有客户启动事务时, Microsoft 系统将自动检查客户确认信息是否存在。 如果不存在，该交易将无法完成。 

**新建 |通知是否发送到确认过程中提供了电子邮件地址的客户？**

无通知过程。

## <a href="" id="policyfaq"></a>政策

**为什么 Microsoft 要求合作伙伴确认客户已接受 Microsoft 云协议？** 

Microsoft 云协议为客户提供适用于 Microsoft 云产品和服务的使用权利、条款和条件，以此保护合作伙伴、客户和 Microsoft 的利益。 

正确执行 Microsoft 云协议可确保对我们的合作伙伴、客户和 Microsoft 进行双向协调, 其中包括但不限于安全性、隐私和数据保护等重要主题。 确认过程可确保透明度和对齐, 并使 Microsoft 可以更好地帮助需要立即响应法规查询的客户。

**完成确认过程需要哪些客户信息？**
需要提供与接受协议的人员相关的以下信息：
-   协议接受日期
-   名字和姓氏
-   电子邮件地址
-   电话号码（可选） 

**新建 |如果我们不能满足截止日期, 会发生什么情况？**

此功能将于 2018 年 8 月 7 日推出，合作伙伴随即可以开始进行接受确认。 从2018年11月7日开始, 合作伙伴中心仪表板用户需要确认客户接受所有新事务的 Microsoft 云协议。 2019年3月22日, 将要求合作伙伴中心 API 用户执行相同操作。

在实施时, 将不允许对现有客户进行新的购买, 或在确认客户验收之前, 不允许对现有订阅进行座位计数。 对于新客户，在确认接受之前，不允许其进行交易。 自动续订不受此要求的影响。

**此要求适用于哪些区域？**

Microsoft 公有云下的所有地区都需要确认对 Microsoft 云协议的客户接受。 国家云（包括 Microsoft Cloud for US Government 和德国 Microsoft 云）不支持客户接受确认。

**哪些合作伙伴负责确认客户接受 Microsoft 云协议？**

2018年11月7日, 云服务提供商 (CSP) 间接提供商和 CSP 直销合作伙伴共同负责确认客户接受, 并可使用合作伙伴中心 API 或合作伙伴中心仪表板执行此操作。 间接提供商可能需要首先从其间接经销商那里收集所需的信息, 然后通过合作伙伴中心 API 或合作伙伴中心仪表板将其提供给 Microsoft。 合同形式, 它是与 Microsoft 签订了绑定合同的间接提供商, 以确保其 CSP 经销商的最终客户已接受 Microsoft 云协议。 

**哪些客户方案需要确认客户接受 Microsoft 云协议？**

当 CSP 合作伙伴要做出以下交易时, 必须确认接受 Microsoft 云协议:
1.  新客户完成新购买
2.  现有客户完成新购买
3.  现有的客户更新对现有订阅的座位计数

**我是否可以确认是否要将现有客户作为新购买或座位添加的内容？**

是。 可以为 Microsoft 云协议涵盖的所有客户确认客户接受情况。

**具有多个 CSP 合作伙伴的客户的要求是什么？**

接受确认由合作伙伴来执行。 如果客户与多个云解决方案提供商合作伙伴进行交易，则每个合作伙伴都必须进行接受确认。

**我是一个 CSP 间接经销商, 使用 CSP 间接提供商。我的职责是什么？**

合同形式, 它是与 Microsoft 签订了绑定合同的间接提供商, 以确保最终客户已接受 Microsoft 云协议。 但是，间接提供商可能会与间接经销商签订单独的协议来收集客户接受信息。 在此情况下，间接提供商将需要首先从其间接经销商处收集所需的客户接受信息，然后通过合作伙伴中心 API 或合作伙伴中心面板将这些信息提供给 Microsoft。 你可以在收到请求时向间接提供商快速提供所需信息以支持此过程。

**我是一个 CSP 间接提供商。我是否需要与我的 CSP 间接经销商进行协调才能准备就绪？**

建议需要从间接经销商处收集所需客户接受信息的间接提供商立即将此要求传达给其间接经销商，以协调此信息的传递和接收。

**我如何向客户说明他们是否应确认接受 Microsoft 云协议？**

正确执行 Microsoft 云协议可确保 Microsoft、合作伙伴和客户通过确保与安全、隐私和数据保护在内的许多重要主题的相互协调, 来保护 Microsoft 及其合作伙伴和客户。 确认过程可确保透明度和对齐, 并使 Microsoft 可以更好地帮助需要立即响应法规查询的客户。

**新建 |如果客户使用单个 CSP 间接提供程序, 但使用多个 CSP 间接经销商完成事务, 则每个 CSP 间接经销商是否需要提供确认信息的 CSP 间接提供程序？**

CSP 间接提供程序负责向 Microsoft 提供客户接受 Microsoft 云协议的确认。 对于每个客户帐户, 一次只能在合作伙伴中心仪表板中记录一次确认。 

CSP 间接提供程序负责确认负责同意 Microsoft 云协议的客户联系人是否已执行此操作, 而不管他们使用的 CSP 间接经销商的数量是多少。

**新建 |谁应作为授权人员确认接受客户组织内的 Microsoft 云协议？**

合作伙伴负责从客户组织内的某位客户接受 Microsoft 云协议。 客户负责确定谁有权提供接受。 

**新建 |哪个版本的 Microsoft 云协议是负责确认客户接受的合作伙伴？合作伙伴是否需要指出客户接受的 Microsoft 云协议版本？**

当向新客户进行 CSP 交易时, 合作伙伴负责使用最新的 Microsoft 云协议。 当使用现有客户续订、更改或完成新交易时, 合作伙伴必须确认接受最新的 Microsoft 云协议。 

如果在续订之前更新 Microsoft 云协议, 则合作伙伴无需让客户立即重新获得 Microsoft 云协议的客户接受, 但必须在此之前再次确认同意 Microsoft 云协议续订。 但是, 你可能希望客户在续订之前接受新条款, 以利用 Microsoft 云协议中的新条款和改进条款。

**新建 |我是一个 CSP 间接经销商。如果在我的公司记录中记录客户对 Microsoft 云协议的接受, 还需要通过合作伙伴中心仪表板或合作伙伴中心 API 提供此信息吗？**

合同形式, 它是与 Microsoft 签订了绑定合同的间接提供商, 以确保最终客户已接受 Microsoft 云协议。 但是，间接提供商可能会与间接经销商签订单独的协议来收集客户接受信息。 在此情况下，间接提供商将需要首先从其间接经销商处收集所需的客户接受信息，然后通过合作伙伴中心 API 或合作伙伴中心面板将这些信息提供给 Microsoft。

**新建 |如果客户不同意 Microsoft 云协议条款, 该怎么办？**

作为 Microsoft 的 CSP 计划中的合作伙伴, 你负责在下订单之前, 客户接受 Microsoft 云协议。 这不是一种新的要求。 出于任何原因, 如果客户不再同意 Microsoft 云协议, 则合作伙伴必须取消与此客户相关联的订阅。

**新建 |经销商能否代表客户提供许可？**

否。

**确认客户已提供接受后, 是否需要为客户重新确认该客户？**

确认客户已接受 Microsoft 云协议后, 此信息将保留在合作伙伴中心的 "客户帐户" 部分中, 并且无需在客户被覆盖的时间段内重新确认接受协议条款。 在提出新订单之前, 只能重新确认接受。

如果在续订之前更新 Microsoft 云协议, 则合作伙伴无需让客户立即重新获得 Microsoft 云协议的客户接受, 但必须在此之前再次确认同意 Microsoft 云协议续订。 但是, 你可能希望客户在续订之前接受新条款, 以利用 Microsoft 云协议中的新条款和改进条款。

**新建 |我不喜欢为 Microsoft 最终客户提供直接联系信息。Microsoft 将如何使用此信息？**

当合作伙伴创建客户租户时, 需要提供客户的主要联系信息。 根据联机服务条款, "Microsoft 将仅处理来自客户的已记录说明的个人数据"。

## <a href="" id="technicalfaq"></a>技术

**合作伙伴如何确认客户接受并向 Microsoft 提供此功能？哪些工具可用于支持此过程？**

从2018年8月7日开始, 有两种方法可帮助合作伙伴: 
1.  合作伙伴可以使用合作伙伴中心 API 以编程方式进行客户接受确认。 
2.  合作伙伴可以使用合作伙伴中心仪表板确认客户接受情况。

**新建 |我是使用合作伙伴中心 API 的 CSP 合作伙伴。需要考虑哪些 API 更改？** 

在这种情况下, API 将更改为的帐户:

-   [https://docs.microsoft.com/partner-center/develop/get-confirmation-of-customer-consent](https://docs.microsoft.com/partner-center/develop/get-confirmation-of-customer-consent) 

-   [https://docs.microsoft.com/partner-center/develop/get-agreement-metadata](https://docs.microsoft.com/partner-center/develop/get-agreement-metadata)

-   [https://docs.microsoft.com/partner-center/develop/confirm-customer-consent](https://docs.microsoft.com/partner-center/develop/confirm-customer-consent)

**如果我难以实现合作伙伴中心 API, 会怎么样？**

可以通过合作伙伴中心面板手动进行接受确认。

**新建 |作为使用合作伙伴中心仪表板或合作伙伴中心 API 的替代方法, 能否改为使用客户确认信息批量上传文件？**

目前, 合作伙伴中心仪表板和合作伙伴中心 API 是唯一接受的方法, 可确认客户接受 Microsoft 云协议。

**新建 |我是使用第三方平台或 "中间件" 的合作伙伴, 它执行将公司的系统与合作伙伴中心连接的功能。需要执行哪些操作？**

建议使用集成了合作伙伴中心 API 的平台的合作伙伴直接与有关其计划和时间线的访问接口进行交流, 以便集成合作伙伴中心 API 功能, 从而自动执行客户确认流程。

**新建 |能否提供用于导入客户全名、电子邮件地址和 Microsoft 云协议接受日期的 API 流示例？**

建议合作伙伴查看合作伙伴中心 API 文档以了解详细信息和示例。

**新建 |如何实现获取向客户显示的最新 Microsoft 云协议, 并将其作为 API 过程的一部分在确认记录中使用？**

当前合作伙伴中心 API 是非常基本的, 不能让合作伙伴以编程方式检索 Microsoft 云协议的副本, 也不会捕获客户在确认过程中接受的 Microsoft 云协议的版本。 

**新建 |对此过程使用合作伙伴中心 API 需要 userID 参数。当订单通过与 Azure AD 不集成的第三方平台时, 如何实现管理此要求？**

在这种情况下, userID 标识合作伙伴租户下的员工, 该员工将提供客户同意的确认。 如果员工的租户下没有相应的 Azure AD 用户帐户, 则建议你为该目的指定特定的用户帐户, 并创建一个过程来单独捕获提供证实. 

**新建 |为什么 Microsoft 要求合作伙伴确认客户同意 Microsoft 云协议, 而不是直接从客户那里获取它？**

客户是从 CSP 合作伙伴 (而不是直接从 Microsoft) 购买的。 为了使用 Microsoft 提供的服务, Microsoft 提供了 Microsoft 云协议中记录的条款。 Microsoft 云分销商协议要求 CSP 合作伙伴在提出订单之前, 先查看 Microsoft 云协议并接受该协议。 

**新建 |我同时使用 API 和合作伙伴中心仪表板 web 用户界面来确认 Microsoft 云协议客户接受。2018年11月7日之后, 有哪些选项可用于确认客户接受？**

这两个选项都可用且可接受, 可用于确认 Microsoft 云协议客户接受。 使用合作伙伴中心仪表板的事务需要在2018年11月7日开始确认。 使用合作伙伴中心 API 的事务需要在2019年3月22日开始确认。
