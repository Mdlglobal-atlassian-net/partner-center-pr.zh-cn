## <a href="" id="processfaq"></a>Process

**确认 Microsoft 云协议的客户验收新流程是什么？**

Microsoft 致力于帮助我们的客户和合作伙伴保持合规。 作为 Microsoft 的云解决方案提供商 (CSP) 计划的合作伙伴，您有责任审阅 Microsoft 的条款和条件，包括可接受用法的 Microsoft 产品和服务与客户的。 若要更好地帮助满足合规性要求的合作伙伴，并以确保透明性，我们启用 CSP 合作伙伴必须确认其客户已接受的 Microsoft 云协议条款的新进程。 截至 2018 年 8 月 7 日，我们引入了新字段在合作伙伴中心仪表板用户界面中也适用于的合作伙伴中心 API 合作伙伴可提供此信息。 最初它将是可选的以提供此确认，但将它从 2018 年 11 月 7 日开始。 

**此要求起何时开始？**

此功能将于 2018 年 8 月 7 日推出，合作伙伴随即可以开始进行接受确认。 开始于 2018 年 11 月 7 日，合作伙伴中心仪表板用户需要确认的所有新事务上的 Microsoft 云协议的客户验收。 在 2019 年 3 月 22 日上合作伙伴中心 API 用户将需要执行相同操作。 API 用户在截止时间已扩展为支持这些其他技术实现的时间。

**新 |Microsoft 是否验证合作伙伴提供的客户信息的准确性？**

Microsoft 验证提供的客户电子邮件地址正确的格式。 它负责的合作伙伴提供准确的信息并更正找到不准确的任何信息。 

**新 |如何可以合作伙伴证明的 Microsoft 云协议的客户的验收是否进行签名，已记录在案、 数字化，或在纸上？**

合作伙伴是负责确保客户已接受 Microsoft 云协议的条款，并管理其自己的进程和记录。 通过使用合作伙伴中心仪表板或合作伙伴中心 API 流程，合作伙伴确认客户已接受 Microsoft 云协议的条款。

**新 |Microsoft 如何检查确认过程已完成的客户？**

从 2018 年 11 月 7，，当 CSP 合作伙伴启动新的或现有客户使用合作伙伴中心仪表板的事务，Microsoft 的系统将自动检查客户确认信息是否存在在合作伙伴中心仪表板。 如果不存在，该交易将无法完成。 

从年 3 月 22，2019，当 CSP 合作伙伴启动事务的使用合作伙伴中心 API 的新的或现有客户，Microsoft 的系统将自动检查客户确认信息是否存在。 如果不存在，该交易将无法完成。 

**新 |向客户确认过程中提供的电子邮件地址发送通知？**

没有通知的进程。

## <a href="" id="policyfaq"></a>策略

**为什么 Microsoft 会要求确认客户已接受 Microsoft 云协议的合作伙伴？** 

Microsoft 云协议保护合作伙伴、 客户和 Microsoft 通过提供客户的使用权限、 条款和条件授予 Microsoft 云产品和服务订单。 

Microsoft 云协议的正确执行通过确保包括的许多重要主题相互对齐方式保护我们的合作伙伴、 客户和 Microsoft，但不是限于安全性、 隐私和数据保护。 确认过程可确保透明度和对齐方式，并允许 Microsoft 为了更好地帮助客户需要立即响应法规的查询。

**哪些客户信息是完成确认过程所必需的？**
需要提供与接受协议的人员相关的以下信息：
-   协议接受日期
-   名字和姓氏
-   电子邮件地址
-   电话号码（可选） 

**新 |如果我们不能满足截止日期，则会发生什么情况？**

此功能将于 2018 年 8 月 7 日推出，合作伙伴随即可以开始进行接受确认。 开始于 2018 年 11 月 7 日，合作伙伴中心仪表板用户需要确认的所有新事务上的 Microsoft 云协议的客户验收。 在 2019 年 3 月 22 日上合作伙伴中心 API 用户将需要执行相同操作。

强制时，新购买的座位数对现有订阅的现有客户或更改将不允许客户验收确认之前禁止。 对于新客户，在确认接受之前，不允许其进行交易。 自动续订不受此要求的影响。

**此要求适用于哪些区域？**

需要在 Microsoft 公有云的所有地区的 Microsoft 云协议的客户验收的确认。 国家云（包括 Microsoft Cloud for US Government 和德国 Microsoft 云）不支持客户接受确认。

**哪些合作伙伴负责确认到 Microsoft 云协议的客户验收？**

2018 年 11 月 7 日，云服务提供商 (CSP) 间接提供商和 CSP 直接帐单合作伙伴都负责确认客户验收，而且可以使用合作伙伴中心 API 或合作伙伴中心仪表板来执行此操作有效。 间接提供商可能需要首先从其间接经销商收集所需的信息，然后将其提供给 Microsoft 通过合作伙伴中心 API 或合作伙伴中心仪表板。 根据合同，它是间接提供商具有与 Microsoft 以确保其 CSP 分销商最终客户已接受到 Microsoft 云协议联系的绑定。 

**哪些客户方案需要确认客户验收的 Microsoft 云协议？**

当 CSP 合作伙伴想要使以下事务时，必须确认 Microsoft 云协议接受：
1.  新客户完成新购买
2.  现有客户完成新购买
3.  现有客户向现有订阅更新的许可席位计数

**可以确认适用于独立于其新购买的现有客户接受或座位新增功能？**

是。 客户验收可以确认包括在 Microsoft 云协议的所有客户。

**对于具有多个 CSP 合作伙伴的客户要求是什么？**

接受确认由合作伙伴来执行。 如果客户与多个云解决方案提供商合作伙伴进行交易，则每个合作伙伴都必须进行接受确认。

**我是 CSP 间接经销商使用 CSP 间接提供商。我的责任是什么？**

根据合同，它是间接提供商具有与 Microsoft 以确保最终用户已接受 Microsoft 云协议联系的绑定。 但是，间接提供商可能会与间接经销商签订单独的协议来收集客户接受信息。 在此情况下，间接提供商将需要首先从其间接经销商处收集所需的客户接受信息，然后通过合作伙伴中心 API 或合作伙伴中心面板将这些信息提供给 Microsoft。 你可以在收到请求时向间接提供商快速提供所需信息以支持此过程。

**我是 CSP 间接提供商。我是否需要与我 CSP 间接经销商，以准备进行协调？**

建议需要从间接经销商处收集所需客户接受信息的间接提供商立即将此要求传达给其间接经销商，以协调此信息的传递和接收。

**如何能我解释一下到我的客户，他们应确认其接受 Microsoft 云协议？**

Microsoft 云协议的正确执行通过确保包括的许多重要主题相互对齐方式来保护 Microsoft 及其合作伙伴和客户，但不是限于安全性、 隐私和数据保护。 确认过程可确保透明度和对齐方式，并允许 Microsoft 为了更好地帮助客户需要立即响应法规的查询。

**新 |如果客户可使用单一的 CSP 间接提供程序，但与多个 CSP 间接经销商完成事务，是否每个 CSP 间接经销商需要 CSP 间接提供商提供的确认信息？**

CSP 间接提供商负责向 Microsoft 提供的 Microsoft 云协议客户验收的确认。 对于每个客户帐户，可以一次在合作伙伴中心仪表板中记录只有一个确认。 

CSP 间接提供商负责进行确认，客户的联系人负责同意 Microsoft 云协议还执行该操作，无论它们与交易 CSP 间接经销商数量是多少。

**新 |谁应该是获得授权的人员，以确认接受客户组织内部的 Microsoft 云协议？**

它是从客户组织内部人员获得 Microsoft 云协议的客户验收合作伙伴的责任。 客户负责确定谁有权提供接受它。 

**新 |哪个版本的 Microsoft 云协议是负责确认客户验收的合作伙伴？是否需要指示哪个版本的 Microsoft 云协议已接受由客户合作伙伴？**

合作伙伴负责进行与新客户的 CSP 事务时使用最新的 Microsoft 云协议。 合作伙伴必须确认接受最新的 Microsoft 云协议时续订、 更改或完成与现有客户的新事务。 

在续订前，将更新为 Microsoft 云协议的情况下，合作伙伴都不需要重新立即获取 Microsoft 云协议的客户验收客户，但必须再次确认同意之前的 Microsoft 云协议为续订。 但是，你可能想要让客户接受新条款之前续订以利用 Microsoft 云协议中的新增和改进的术语。

**新 |我是 CSP 间接经销商。如果我在我的公司的记录中记录的 Microsoft 云协议的客户验收，是否需要也提供此信息通过合作伙伴中心仪表板或合作伙伴中心 API？**

根据合同，它是间接提供商具有与 Microsoft 以确保最终用户已接受 Microsoft 云协议联系的绑定。 但是，间接提供商可能会与间接经销商签订单独的协议来收集客户接受信息。 在此情况下，间接提供商将需要首先从其间接经销商处收集所需的客户接受信息，然后通过合作伙伴中心 API 或合作伙伴中心面板将这些信息提供给 Microsoft。

**新 |如果我的客户到 Microsoft 云协议的条款不一致？**

作为 Microsoft 的 CSP 计划中的合作伙伴，您有责任在之前的订单的 Microsoft 云协议的客户的验收。 这不是新要求。 出于任何原因，如果客户无法再同意通过 Microsoft 云协议，然后合作伙伴必须取消与此客户关联的订阅。

**新 |分销商可以提供代表客户的许可？**

否。

**一旦我确认客户提供了接受，是否需要重新确认其客户将来？**

一次确认客户已接受 Microsoft 云协议，此信息保留在合作伙伴中心中的客户帐户部分并不需要再次确认接受客户涵盖的时间的持续时间协议的条款。 仅必须再次确认接受之前放置新订单。

在续订前，将更新为 Microsoft 云协议的情况下，合作伙伴都不需要重新立即获取 Microsoft 云协议的客户验收客户，但必须再次确认同意之前的 Microsoft 云协议为续订。 但是，你可能想要让客户接受新条款之前续订以利用 Microsoft 云协议中的新增和改进的术语。

**新 |我不愿意向 Microsoft 提供我的最终客户直接联系信息。Microsoft 将如何使用此信息？**

合作伙伴可以创建客户租户，它则需要为客户提供主要的联系信息。 每个联机服务条款，"Microsoft 将处理上有案可稽的说明，从客户的个人数据。"

## <a href="" id="technicalfaq"></a>技术

**如何合作伙伴确认客户接受并将其提供给 Microsoft？哪些工具是可用于支持此进程？**

可从 2018 年 8 月 7 日开始的两种方法来帮助合作伙伴： 
1.  合作伙伴可以使用合作伙伴中心 API 以编程方式进行客户接受确认。 
2.  合作伙伴可以使用合作伙伴中心仪表板来确认客户接受。

**新 |我是 CSP 合作伙伴使用合作伙伴中心 API。哪些 API 更改需要考虑我？** 

在这种情况下，若要为帐户的 API 更改是：

-   [https://docs.microsoft.com/en-us/partner-center/develop/get-confirmation-of-customer-consent](https://docs.microsoft.com/en-us/partner-center/develop/get-confirmation-of-customer-consent) 

-   [https://docs.microsoft.com/en-us/partner-center/develop/get-agreement-metadata](https://docs.microsoft.com/en-us/partner-center/develop/get-agreement-metadata)

-   [https://docs.microsoft.com/en-us/partner-center/develop/confirm-customer-consent](https://docs.microsoft.com/en-us/partner-center/develop/confirm-customer-consent)

**如果遇到实现合作伙伴中心 API？**

可以通过合作伙伴中心面板手动进行接受确认。

**新 |作为使用合作伙伴中心仪表板或合作伙伴中心 API 的替代方法，可以批量上传客户确认信息的文件而是？**

在此期间，合作伙伴中心仪表板和合作伙伴中心 API 是唯一的接受的方法提供的 Microsoft 云协议的客户验收的确认。

**新 |我使用第三方平台或"中间件，"执行功能将我的公司的系统连接到合作伙伴中心的合作伙伴。我需要做什么？**

建议合作伙伴使用平台集成合作伙伴中心 API 的直接咨询有关其计划和时间线的提供程序进行集成的合作伙伴中心 API 功能，可自动执行客户确认过程。

**新 |您用于导入客户全名、 电子邮件地址和 Microsoft 云协议接受日期提供 API 流示例**

建议合作伙伴将查看合作伙伴中心 API 文档，了解详细信息和示例。

**新 |如何获取最新的 Microsoft 云协议来向客户显示并在确认记录中使用我 API 过程的一部分？**

当前合作伙伴中心 API 是非常基本的和不提供用于以编程方式检索一份 Microsoft 云协议的合作伙伴也不会捕获版本的 Microsoft 云协议接受客户作为确认的一部分。 

**新 |对于此过程使用合作伙伴中心 API 需要用户 Id 参数。通过不与 Azure AD 集成的第三方平台下的订单时，如何管理此要求？**

用户 Id 在这种情况下标识下提供的确认客户同意合作伙伴租户的员工。 如果员工没有相应的 Azure AD 用户帐户下合作伙伴的租户，则建议你为该目的指定的特定用户帐户并创建一个单独捕获的员工提供标识的过程确认。 

**新 |为什么 Microsoft 会要求确认客户同意向 Microsoft 云协议而不是直接从客户获取的合作伙伴？**

客户从 CSP 合作伙伴，不能直接从 Microsoft 购买。 服务使用的提供了 Microsoft，Microsoft 提供的条款的 Microsoft 云协议中所述。 Microsoft 云经销商协议要求 CSP 合作伙伴拥有查看并接受 Microsoft 云协议之前放置一个订单的客户。 

**新 |我使用 API 和合作伙伴中心仪表板 web 用户界面来确认 Microsoft 云协议客户接受。后于 2018 年 11 月 7 日，我有哪些选项来确认客户接受？**

这两个选项是可用和可接受的确认 Microsoft 云协议客户的验收。 使用合作伙伴中心仪表板的事务将需要确认存在从 2018 年 11 月 7 日开始。 使用合作伙伴中心 API 的事务将需要确认存在从 2019 年 3 月 22 日。
