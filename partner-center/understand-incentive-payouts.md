#<a name="incentive-payouts-overview"></a>奖励付款概述

**适用范围**

-  合作伙伴中心

奖励管理员和用户可大概了解他们已经或即将从奖励计划收到的收益和付款。   

##<a name="summary-information-on-earnings-and-payout"></a>关于收益和付款的摘要信息

奖励用户和奖励管理员可以在**付款摘要**中查看他们在所参与的奖励计划中的财务状况。 你可以根据需要详细分析每个摘要信息区域。 

在摘要页面，你可以对你获得付款的月份以及计划类型进行分析的趋势进行了解。 反之，你也可以按计划和位置查看你所赚取的款项。 这可以使你了解你在哪里获得了成功以及在哪里应投入更多精力进行营销和销售。 “付款”摘要页面显示收益和付款。奖励数据每 48 小时刷新一次。 图表反映了最新信息。 

“摘要”页面上使用了三个关键指标：

|**关键指标**   |**定义**   |
|---------------------|:-----------------|
|到目前为止的付款总额。|Microsoft 已向你支付的的金额|
|未付款的收益|你已赚得但尚未计划付款的金额|
|待付款| 计划付款的金额|


分析每个奖励计划的收益和付款。 跟踪收益和付款之间的时间线将有助于规划未来的资源需求等。 你还可以了解奖励计划中哪些地点更成功或更活跃。 

## <a name="payment-trends"></a>付款趋势

摘要的付款趋势部分按你参加的计划显示你的付款。 付款趋势视图是用页面上的数据筛选器控制的。 付款计划是由计划设计决定的。 请参阅你的计划指南了解更多详细信息。 

## <a name="total-earnings-map"></a>总收益地图

总收益地图向你展示你在哪些地区盈利。 地图上反映了你进行交易的位置。 这意味着如果你在美国进行了销售，地图上会反映出来。 此视图与计划无关。 

## <a name="product-earnings-summary"></a>产品收益摘要

产品/计划收益和付款表显示每个计划/产品的总收益金额、未付款的收益、待付款和到期支付总额的细分。 



## <a name="analyze-incentives-earnings"></a>分析奖励收益

了解贵组织在哪里有机会通过奖励计划获取更多收益，以及你已经在哪里获得了付款。 此数据清楚地表明了你在哪里获得了收益但尚未获得付款。  了解哪些计划和活动是最成功的，并找出你在哪些位置正在获得收益。 按照计划，你可以查看总体收益潜力如何、你可以参与计划的时间段以及计划的到期时间。 收益的每个计划和类型将具有不同的要求及开始和结束日期。 

## <a name="earning-trends"></a>收益趋势 

收益趋势图显示开始或更改你的收益的常规方向。 这一图形视图由日期范围选择器控制，将根据你所选择的日期范围显示数据。 

### <a name="earning-trends-category-definitions"></a>收益趋势类别定义

|   |**定义**   |
|----------------|:--------------|
|已付款的收益|成功付款的收益|
|未付款的收益|尚未计划付款的收益|

### <a name="earning-types"></a>收益类型

有两种收益类型：

- 退款
- 调整

## <a name="unpaid-earnings-summary"></a>未付款的收益摘要

未付款的收益表显示尚未计划付款的未付款收益的摘要。 按计划/产品和收益类型进行划分。 你可以看到收益阶段开始日期以及收益结束的结束日期。 收益活动日期是收益成熟到可以付款的时间。 如果收益可以到期并且未支付，则到期日期适用。

## <a name="payments-and-statements"></a>付款和明细表

付款和明细表页面上的地图显示基于位置的交易的付款。 例如，如果你在美国进行了销售并收到了其付款，这将显示在地图上。 此视图是你所涉及的所有计划的汇总。

## <a name="payments"></a>付款

付款表显示你在过去 12 个月中进行的所有付款。 按计划和收益类型进行细分。 你可以看到收益阶段的开始和完成时间。 当付款发送到银行时，你将会看到付款日期和付款 ID。

## <a name="statements"></a>明细表

当付款发送到银行时会生成明细表。 单击**视图**可查看付款的详细信息。 

下载交易会显示与该月发生的支付关联的交易和收益的详细视图。 

下载付款详细信息会细分付款。 如果一个付款拆分为多个付款，你可以在此详细视图中查看它是如何拆分的。
