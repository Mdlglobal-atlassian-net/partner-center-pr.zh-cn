# <a name="transition-from-cloud-solution-provider-csp-direct-bill-partner-to-csp-indirect-reseller"></a>从云解决方案提供商 (CSP) 计划直接计费合作伙伴过渡到云解决方案提供商计划间接经销商 

>[!Note] 
>本文适用于已决定过渡到间接经销商直接帐单合作伙伴。 

你将能够在使用现有的直接帐单租户的间接分销商程序中进行注册。 

## <a name="get-started"></a>立即开始行动

1. 请确保你的合作伙伴配置文件在合作伙伴中心和 MPN ID 是最新。

2. 登录到合作伙伴中心全局管理员身份直接帐单租户正在转换为间接经销商。

![概述](images/direct/direct1.png) 

3. 查看你注册窗体上的合作伙伴详细信息。

![立即注册](images/direct/direct2a.png) 

4. 选择立即注册。 间接经销商业务将使用相同的 AAD 租户用于直接业务。

>[!Note]
>最初，这项新转换功能将年 12 月周年日期可供具有年 9 月的合作伙伴。 如果您不需要使用"t 具有年 9 月和年 12 月之间的周年日期，不会看到这一次的功能。 为合作伙伴启用该功能后，将更高版本通知周年日期 2018 年 12 月之后的合作伙伴。


5. 在批准你的注册时再次登录到合作伙伴中心。

>[!Note]
> 通常立即批准时，可能需要最多 5 个工作日。 批准后，将收到通知到具有指定下注册窗体中的主要联系人的电子邮件地址。 您还可以检查您的注册状态下**设置**→**合作伙伴设置**→**合作伙伴配置文件**→ 节目信息。 

6. 在你**概述**页上，您将看到间接经销商协议。 选择**接受并继续操作**。 这样，间接经销商功能。

接受间接经销商协议后，请注意，你的合作伙伴配置文件标识作为**同时**直接帐单和间接经销商。

![间接经销商协议](images/direct/direct3.png)

>[!Important] 
>一旦注册为使用新功能间接经销商，没有回滚到直接帐单-只有租户的选项。 请确保您全面评估你之前注册为间接分销商的业务需求。

## <a name="while-you-transition-from-direct-to-indirect-reseller"></a>虽然从直接转换到间接经销商

在此阶段中将继续管理直接客户的订阅的要求包括计费过程。 你可以开始接受来自间接提供商和作为间接经销商的客户。

![是一个直接的帐单和间接经销商](images/direct/direct4.png)

## <a name="find-an-indirect-provider"></a>查找间接提供商

注册后，间接提供商的链接将显示在左侧的导航栏。 为间接分销商将与间接提供商然后可以处理您的付费，为你的客户购买产品和支持基础结构建立关系。

不同的间接提供商会提供不同的支持和服务，因此你应该评估你所在地区中的提供商，以确定哪些提供商最能够满足你的需求。 通常，大多数提供商将： 

- 为你提供技术培训和援助
- 帮助你推销你的产品和服务 
- 管理你融资和信用额度的条件 

搜索的官方列表[Microsoft 间接提供商](https://partnercenter.microsoft.com/partner/find-a-provider)。

了解详细信息，读取[与间接提供商合作](indirect-reseller-tasks-in-partner-center.md)

## <a name="accept-a-partnership-invitation-from-your-indirect-provider"></a>接受从间接提供商合作关系邀请

找到要与合作伙伴的间接提供程序后，使用合作伙伴中心中的间接提供程序建立合作关系。

您选择的间接提供程序将向你发送电子邮件中将转到其邀请合作伙伴中心中的合作关系邀请链接。 请确保全局系统管理员登录到合作伙伴中心，并遵循该邀请链接。 在接受邀请后，将间接提供商列表中显示提供程序的名称。

## <a name="acquire-new-customers-as-indirect-reseller"></a>为间接经销商获得新客户

你和你间接提供商需要能够与客户的分销商关系。 这使您能够管理客户的订阅和服务代表其执行。 若要获取已有 Azure AD 租户的新客户，可以邀请，客户可以在同一时间与您以及您的提供程序建立的分销商关系。 

若要创建的间接经销商邀请：

1. 选择**间接提供商**从合作伙伴中心左导航 

2. 选择**邀请新客户**以邀请客户同时与你和间接提供商同时建立经销商关系。 提供程序需要具有与你客户的分销商关系，因此客户想要购买新订阅或向现有订阅添加新的许可证时，他们可以提交代表客户的订单。

3. 在下一页上，查看草稿电子邮件。 您可以在电子邮件，打开草稿消息也可以将消息复制到剪贴板并将其粘贴到电子邮件中。

4. 编辑要假定您需要但务必要包含链接，因为它个性化客户直接连接到你的帐户和提供程序的帐户的电子邮件中的文本。 然后选择**完成**。

5. 客户准许你和你的提供商成为其备案经销商后，你将具有管理员权限，可以代表他们管理其订阅、许可证和用户，并且你的间接提供商将能够代表他们提交订单。
6. 若要管理客户的帐户、服务、用户和许可证，请选择客户名称旁边的向下箭头展开客户的记录。 

与直接帐单伙伴不同间接经销商不能在合作伙伴中心创建新客户的 Azure AD 租户。 您的提供程序将创建租户，并将指定你为此客户的间接经销商。 这可确保客户将出现在您在合作伙伴中心中的客户列表。

>[!Note]
>你将不能使用直接帐单功能来创建作为间接经销商获取的客户的购买。

1. 选择**间接提供商**从合作伙伴中心左导航

2. 选择**邀请新客户**邀请客户可以在同一时间建立与您以及间接提供商的分销商关系。 提供程序需要具有与你客户的分销商关系，因此客户想要购买新订阅或向现有订阅添加新的许可证时，他们可以提交代表客户的订单。

3. 在下一页上，查看草稿电子邮件。 您可以在电子邮件，打开草稿消息也可以将消息复制到剪贴板并将其粘贴到电子邮件中。 

4. 编辑要假定您需要但务必要包含链接，因为它个性化客户直接连接到你的帐户和提供程序的帐户的电子邮件中的文本。 然后选择**完成**。  

5. 客户准许你和你的提供商成为其备案经销商后，你将具有管理员权限，可以代表他们管理其订阅、许可证和用户，并且你的间接提供商将能够代表他们提交订单。

6. 若要管理客户的帐户、服务、用户和许可证，请选择客户名称旁边的向下箭头展开客户的记录。


## <a name="managing-your-direct-bill-customers-and-your-indirect-reseller-customers"></a>管理直接帐单客户和你间接经销商的客户

以不同的方式管理你对客户直接开具帐单和你间接经销商的客户。 

**直接对客户开具帐单 （不会作为间接经销商处执行的操作）**

- 创建订单的产品
- 管理 Azure 保留项
- 管理其订单历史记录
- 购买软件
- 直接向客户收取费用

**间接经销商的客户**

- 间接提供商为您的客户订单的产品
- 管理客户的许可证和用户
- 处理订阅续订

**若要标识作为直接帐单合作伙伴获取的客户**

1. 选择**客户**

2. 选择要查看其详细信息

3. 如果此客户是一个获取作为直接帐单合作伙伴，您将看到的选项**添加**或**查看产品**，你将看到其订阅。

4. 如果客户具有与你的间接经销商关系，这些选项将不可用。

## <a name="what-happens-when-your-direct-bill-capabilities-have-been-restricted"></a>当你直接帐单的功能已被限制时，会发生什么情况

您已决定过渡到间接经销商和具有双重注册，如直接帐单和间接经销商，或不进行了明确界定尚未注册为间接经销商，是否直接帐单合作伙伴不符合新要求用于 CSP 直接帐单合作伙伴计划将在时得到通知 microsoft 他们直接帐单的能力会受到限制。 

在你**合作伙伴配置文件**下**编程信息**你将看到您的 Microsoft 云解决方案提供商状态已更改为**受限**。

## <a name="affected-direct-bill-capabilities"></a>受影响的直接帐单功能

在直接帐单功能受到限制，请就不再可以为您的客户在合作伙伴中心进行新购买。 这包括： 

- Azure 订阅
- 基于许可证的订阅
- 将新的加载项添加到现有的基于许可证的订阅
- 购买软件和保留产品，例如软件订阅、 永久软件和 Azure 保留虚拟机实例的一次性的商品。

也不能使用 CSP 计划下的 Azure 合作伙伴共享的服务产品/服务购买新的 Azure 订阅。 

除了新的购买不能访问合作伙伴中心中的以下直接帐单功能： 
- 无法创建新的客户租户。 **创建客户**下**客户**在合作伙伴中心将不可用。 

- 无法生成邀请客户请求直接分销商关系。 **请求经销商关系**下**客户**在合作伙伴中心将不可用。 

>[!Note] 
>如果已注册为间接经销商在直接帐单合作伙伴租户，你就能够将邀请发送给客户请求而不是直接帐单间接经销商关系。 

- 无法创建新的沙盒租户。 每个直接帐单合作伙伴租户可以创建一个沙盒租户为了直接帐单 API 集成。 如果你之前未创建一个您将不能直接帐单合作伙伴功能已被限制之后执行此操作。 

## <a name="whats-not-affected"></a>什么是不受影响

不会影响现有直接帐单订阅。 他们仍将有效，并且自动续订。 将继续之前取消它们，由 Microsoft 直接计费。 

继续按以下方式管理现有的订阅： 

- 挂起的现有订阅 

- 调整现有基于许可证的预订的座位数 

- 调整到订阅的现有外接程序的座位数。 注意：无法向现有订阅添加新外接程序，因为外接程序将被视为新的购买。 

- 部署新的 Azure 资源和管理现有的 Azure 订阅下的现有 Azure 资源。 这包括通过 Azure marketplace 和 Visual Studio 订阅提供的产品。 

## <a name="enroll-for-indirect-reseller-incentives"></a>注册进行间接经销商激励 

注册成功后作为间接经销商上现有的直接帐单合作伙伴租户后，将收到 30 天内注册间接经销商激励的邀请。 邀请基于合作伙伴当前与 CSP 合作伙伴租户相关联的 MPN 帐户。 将与合作伙伴的 MPN 帐户关联的电子邮件地址发送的邀请。 

您还有资格注册的具有该相同的合作伙伴租户的直接帐单激励程序。 你必须单独管理程序。

另请参阅[成为间接分销商的其他信息](https://assetsprod.microsoft.com/csp-directbill-to-indirect-transition.pdf)并[间接经销商直接帐单常见问题解答](https://assetsprod.microsoft.com/mpn/direct-bill-partner-faq.pdf)。
