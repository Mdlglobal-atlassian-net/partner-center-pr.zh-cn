# <a name="partner-center-for-microsoft-cloud-germany"></a>德国 Microsoft 云合作伙伴中心

**适用于**

-  德国 Microsoft 云合作伙伴中心

## <a name="microsoft-cloud-germany-meeting-german-data-protection-requirements"></a>德国 Microsoft 云：满足德国数据保护要求 

德国 Microsoft 云专为满足严格的德国数据保护要求而设计，并包含德国数据受托人，该受托人控制按照德国法律对客户数据进行物理和逻辑访问。 所有客户数据和以及对客户数据进行处理或控制访问的系统均位于德国数据中心。 在德国，这些数据中心之间的网络都是隔离且专用的。

德国 Microsoft 云提供 [Microsoft Azure](https://go.microsoft.com/fwlink/?linkid=847992)、Office 365 和 Microsoft Dynamics 的单独实例，它们从德国的数据中心提供。 因此，欧盟 (EU) 和欧洲自由贸易联盟 (EFTA) 中的商业客户在遵守适用的德国法律法规以及重要国际标准的前提下，可以存储和管理客户数据。 未经德国数据受托人或客户的批准和监督，Microsoft 人员无法访问客户数据。

有关详细信息，请参阅[德国 Microsoft 云数据表](http://download.microsoft.com/download/6/1/3/613C9ECB-9167-4EF5-B131-3BAD8D8A126C/Microsoft_Cloud_Germany_Datasheet.pdf)。

## <a name="partner-center-for-microsoft-cloud-germany-selling-german-cloud-solutions-in-eu-and-efta"></a>德国 Microsoft 云合作伙伴中心：在欧盟和欧洲自由贸易联盟内销售德国云解决方案

注册德国 Microsoft 云的云解决方案提供商 (CSP) 后，你将使用合作伙伴中心向欧盟和欧洲自由贸易联盟国家/地区内的客户销售德国云特定的解决方案和服务。 

>**注意**<br>
在本文发表时，德国 Microsoft 云合作伙伴中心尚未提供下列合作伙伴中心功能（自发表以来，一些功能可能已添加到合作伙伴中心，此列表可能已过时）：

- 查找合作伙伴
- 业务配置文件
- 引荐
- 许可证分析
- 客户洞察
- 通知中心
- 间接经销商/提供商模型和设置
- 检查服务运行状况
- 独立软件供应商 (ISV) 注册
- 客户特殊资格认定

为遵守管理个人数据的收集和使用的德国法律法规以及重要的国际标准，使用特定服务门户管理以下合作伙伴中心任务。 

功能 | 在何处完成任务
:--- | :---
合作伙伴用户管理 | Office 365 管理中心或德国 Microsoft 云的 Azure 管理门户
客户用户管理 | Office 365 管理中心或德国 Microsoft 云的 Azure 管理门户
重置用户密码 | 德国 Microsoft 云的 Azure 管理门户
许可证管理 | Office 365 管理中心或德国 Microsoft 云的 Azure Active Directory 许可门户

有关管理用户和许可证的信息，请参阅[德国 Microsoft 云合作伙伴中心中的用户和许可证管理](user-management-in-partner-center-for-microsoft-cloud-germany.md)。

有关注册德国 Microsoft 云的云解决方案提供商的信息，请参阅[注册德国 Microsoft 云的云解决方案提供商](enroll-in-csp-for-microsoft-cloud-germany.md)。

## <a name="frequently-asked-questions-about-the-csp-program-for-microsoft-cloud-germany"></a>有关 Microsoft 云德国程序云解决方案提供商计划的常见问题解答

使用这些常见问题解答了解有关 Microsoft 云德国云解决方案提供商 (CSP) 计划的详细信息。 你还可以联系你的帐户管理员以了解更多信息。 

**什么是 Microsoft 云德国？** 

Microsoft 云德国整合了业内领先的 Microsoft 云服务实例与德国境内的独特系统管理架构和专用网络。 在数据受托人这一独特模型下，由一家独立的德国公司控制对客户数据的访问，并且除了客户指定或德国法律要求的第三方外，不与任何其他第三方共享客户数据。 这也称为数据主权。 

**谁有资格参与 Microsoft 云德国 CSP 计划？**

欧盟 (EU)、欧洲自由贸易联盟 (EFTA) 和英国 (UK) 的合作伙伴有资格参与 Microsoft 云德国 CSP 计划，并可以向欧盟、欧洲自由贸易联盟和英国的客户销售产品和服务。 

**商业 CSP 是现有的合作伙伴计划。 Microsoft 云德国 CSP 计划有何不同之处？**

Microsoft 云德国 CSP 计划与商业 CSP 之间存在以下不同点： 
-   资格：Microsoft 云德国 CSP 计划仅适用于欧盟、欧洲自由贸易联盟和英国的合作伙伴。 
-   加入：合作伙伴需要从商业 CSP 单独加入 Microsoft 云德国 CSP 计划。 
-   许可：Microsoft 云德国 CSP 计划的许可文档将会更新，以反映数据受托人的角色。 下文说明了有关许可更改的更多信息。 
-   合作伙伴中心功能：为了满足数据主权要求，将不会为 Microsoft 云德国 CSP 计划启用合作伙伴中心的某些功能。 合作伙伴可以改用 Microsoft Online 门户 (MOP)、Azure Active Directory (Azure AD) 或 Graph API。 

**Microsoft 云德国 CSP 计划许可与商业 CSP 许可有何不同？**

Microsoft 云德国 CSP 计划的 Microsoft 云分销商协议 (MCRA) 和 Microsoft 云经销商协议 (MCDA) 与商业 MCRA 和商业 MCDA 之间存在以下不同点： 
-   前者要求客户接受适用于德国云的 Microsoft 云协议 (MCA) 以及适用于德国云的在线服务补充条款 
-   前者对客户数据要求进行了若干修改。适用于德国云的 MCA 与商业 MCA 之间存在以下两个不同点： 
-   前者包括在线服务补充条款，涉及 Microsoft 针对数据驻留、数据受托人角色以及客户数据的访问所做的其他数据保护承诺 
-   前者包括数据受托人协议。 补充条款的附件 1 包含客户数据受托人协议，T-Systems 经由该协议针对客户数据的控制和访问直接向客户做出承诺 

**销售 Microsoft 云德国 CSP 计划需要哪些资格？**

销售 Microsoft 云德国 CSP 计划所需的资格标准与商业 CSP 的资格标准相同，且同时对直接合作伙伴和间接分销商开放。 如果你已被批准成为商业 CSP 计划的间接分销商，则不需要再指定为 Microsoft 云德国 CSP 计划的间接分销商。 

**作为现有合作伙伴，我需要怎么做才能销售 Microsoft 云德国 CSP 计划？**

合作伙伴需要加入 Microsoft 云德国 CSP 计划。 （Microsoft 云德国的加入与商业 CSP 的加入相互独立）。 加入后，你可以在 Microsoft 云德国内创建客户并交易产品或服务。 如果你利用的是 API，请查看下方有关 API 的详细信息。 

**我还不是云解决方案提供商合作伙伴。 那么我可以在哪里找到有关如何成为云解决方案提供商合作伙伴的详细信息？**

请访问 MPN 门户以详细了解适用于合作伙伴的注册模型。 

**Microsoft 云德国 CSP 计划的客户定价是否与商业 CSP 的客户定价相同？**

不相同。 提供 Microsoft 云德国的独特价值所需的额外成本会反映到定价中。 

**Microsoft 云德国 CSP 计划的合作伙伴折扣模型是否与商业 CSP 的合作伙伴折扣模型相同？**

是。 Microsoft 云德国 CSP 计划的合作伙伴折扣模型与商业 CSP 的合作伙伴折扣模型相同。 

**与商业 CSP 相比，Microsoft 云德国 CSP 计划的合作伙伴奖励机制是否有任何不同？**

没有。 Microsoft 云德国云解决方案提供商计划的合作伙伴奖励机制与商业云解决方案提供商计划的合作伙伴奖励机制相同。 若要了解有关伙伴奖励机制的详细信息，请访问 MPN 门户以了解最新信息。 

**与商业 CSP 相比，Microsoft 云德国 CSP 计划的计费方式是否有任何变化？**
没有。 Microsoft 云德国云解决方案提供商计划的计费标准和要求与商业云解决方案提供商计划的计费标准和要求相同。 若要了解有关计费方式的详细信息，请参阅 MPN 门户上的“资源”部分。 

**Microsoft 云德国 CSP 计划的支持要求是否与商业 CSP 的支持要求相同？**

是。 若要了解有关支持的详细信息，请参阅 MPN 门户上的“资源”部分。 

**我可以向哪些客户细分市场进行销售？**

一经批准为 CSP MCD 的合作伙伴，你就可以向欧盟、欧洲自由贸易联盟和英国的任何商业客户进行销售。 

**Microsoft 云德国 CSP 计划中将会推出哪些产品/服务？**

计划启动时将推出 Microsoft Office 365 Germany 和 Microsoft Azure Germany。 Microsoft Dynamics 365 Germany (CRMOL) 计划于 2017 年第 2 季度推出。 其他服务一经推出，我们会立即将其添加到计划中。 

**Microsoft 云德国 CSP 计划是否提供全部 Azure 服务？**

目前已推出基础结构即服务 (IaaS)、基础平台以及数据服务。 路线图正在不断更新中，并且我们将会在合作伙伴中心提供商业 CSP 文档等相关文档（需要进行身份验证）。
